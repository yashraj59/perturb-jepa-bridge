# Insight Brief 229: Count Likelihood Helps Top-DE But Not Learned Decoders Yet

## Scope

Family O started a separate synthetic-only count-aware perturbation path. It does not replace Family N's train-only matched-mean distillation.

## Count Audit

Synthetic-lite has raw count-like RNA values in `observed_counts`, generated by gamma-Poisson sampling with dropout. The pseudo-count fallback was not used.

- zero fraction: `0.2026`;
- count mean/variance: `107.9808` / `49344.0223`;
- mean library size: `13821.55`;
- log mean-variance correlation: `0.9593`;
- MoM dispersion median: `1.4854`.

## Main Result

The train-only condition count table is the useful Family O signal:

- Poisson NLL: `48.4387`;
- NB NLL with train-only gene-wise dispersion: `4.9933`;
- program recovery: `0.7433`;
- logFC correlation: `0.7562`;
- top50 overlap: `0.6392`;
- direction accuracy: `0.7679`;
- pseudobulk correlation: `0.8815`.

Compared with Family N's train-only expression table, count-scale means improve logFC, top50, direction, and pseudobulk, but slightly reduce program recovery (`0.7433` vs `0.7520`).

## Learned Models

The no-batch count MLPs passed leakage controls but did not beat the table.

- Poisson MLP: program `0.4855`, logFC `0.4293`, top50 `0.5433`.
- NB MLP: program `0.5770`, logFC `0.4057`, top50 `0.5342`.

NB dispersion improved likelihood calibration, not held-out biological recovery.

## Decision

No promotion and no Family N replacement. If Family O continues on this exact split, run only one targeted hybrid NB + program/top-DE auxiliary trial. Broader count-model work should wait for no-exact-key or held-out-condition evaluation.
